# UML Class Diagram — Real Estate AutoML System

## Access Modifier Key

| Symbol | Meaning |
|--------|---------|
| `+` | Public |
| `-` | Private |
| `#` | Protected |

---

## Class Descriptions

### 1. Class: `User`
**Responsibility:** Represents the Real Estate Professional interacting with the system.

**Attributes:**
- `-userId: String`
- `-username: String`
- `-passwordHash: String`
- `-role: String` *(e.g., "Broker", "Analyst")*

**Methods:**
- `+login(credentials): Boolean`
- `+uploadDataset(filePath): Dataset`
- `+requestPrediction(propertyData): float`
- `+viewExplanation(): void`

---

### 2. Class: `AuthManager`
**Responsibility:** Handles security and access control (Process 1.0).

**Attributes:**
- `-activeSessions: Dictionary`
- `-secretKey: String`

**Methods:**
- `+verifyCredentials(user, pass): Boolean`
- `+generateToken(userId): String`
- `+logout(userId): void`

---

### 3. Class: `Dataset`
**Responsibility:** Holds the raw and cleaned data (Process 2.0).

**Attributes:**
- `-rawData: DataFrame`
- `+targetVariable: String`
- `-isCleaned: Boolean`
- `-qualityScore: float`

**Methods:**
- `+loadFromCSV(path): void`
- `+preprocess(): void` *(Handles missing values, encoding, scaling)*
- `+getTrainingSplit(): tuple`
- `+getQualityScore(): float`

---

### 4. Class: `AutoMLTrainer`
**Responsibility:** Manages the training loop and model selection logic (Process 3.0).

**Attributes:**
- `-modelList: List<MLModel>`
- `-performanceLog: Dictionary`
- `-algorithms: List<String>`

**Methods:**
- `+trainAll(dataset): void`
- `+evaluateModels(metric): void`
- `+selectBestModel(): MLModel`

---

### 5. Class: `MLModel` *(Abstract Base Class)*
**Responsibility:** A generic template for any machine learning algorithm.

**Attributes:**
- `#modelID: String`
- `#hyperParameters: Dictionary`
- `+accuracyScore: float`
- `#algorithmName: String`

**Methods:**
- `+train(features, labels): void` *{abstract}*
- `+predict(features): float` *{abstract}*
- `+getMetrics(): Dictionary`

**Concrete Subclasses:**

#### 5a. Class: `RegressionModel` *(extends MLModel)*
**Responsibility:** Implements regression using Linear Regression.

**Attributes:**
- `-algorithm: LinearRegression`

**Methods:**
- `+train(features, labels): void`
- `+predict(features): float`

#### 5b. Class: `ClassificationModel` *(extends MLModel)*
**Responsibility:** Implements classification using Random Forest.

**Attributes:**
- `-algorithm: RandomForest`

**Methods:**
- `+train(features, labels): void`
- `+predict(features): float`

---

### 6. Class: `ModelRegistry`
**Responsibility:** Handles saving and loading trained models (Data Store D3).

**Attributes:**
- `-storagePath: String`
- `-registryDB: Database`

**Methods:**
- `+saveModel(modelObject): Boolean`
- `+loadModel(modelID): MLModel`
- `+getMetadata(modelID): Dictionary`
- `+listModels(): List<MLModel>`

---

### 7. Class: `Explainer`
**Responsibility:** Generates insights and SHAP-based explanations for the user (Process 5.0).

**Attributes:**
- `-explainerType: String` *(e.g., "SHAP", "LIME")*
- `-shapValues: Dictionary`
- `-featureNames: List<String>`

**Methods:**
- `+generateReport(prediction, model): JSON`
- `+getFeatureImportance(model): Dictionary`
- `+computeSHAP(model, data): Dictionary`
- `+getRecommendations(): List<String>`

---

## Relationships

| Relationship | Type | Description |
|---|---|---|
| `RegressionModel` → `MLModel` | **Inheritance** | RegressionModel is a concrete implementation of the abstract MLModel |
| `ClassificationModel` → `MLModel` | **Inheritance** | ClassificationModel is a concrete implementation of the abstract MLModel |
| `AutoMLTrainer` ◆→ `Dataset` | **Composition** (1 to 0..*) | AutoMLTrainer owns Dataset; if the trainer is destroyed, the active data context is gone |
| `ModelRegistry` ◇→ `MLModel` | **Aggregation** (1 to 0..*) | ModelRegistry stores MLModels; models can exist independently in memory before saving |
| `User` → `AutoMLTrainer` | **Association** | User triggers dataset uploads which feed the trainer |
| `User` → `Explainer` | **Association** | User views prediction explanations generated by the Explainer |
| `User` ··→ `AuthManager` | **Dependency** | User depends on AuthManager to authenticate |
| `Explainer` ··→ `MLModel` | **Dependency** | Explainer reads the trained MLModel to compute SHAP values |

---

## Design Decisions

1. **Abstract `MLModel`** — Using an abstract base class allows the `AutoMLTrainer` to train and evaluate any algorithm uniformly, without knowing the concrete implementation. New algorithms (e.g., XGBoost) can be added by simply subclassing `MLModel`.

2. **Composition for `AutoMLTrainer` ↔ `Dataset`** — The trainer controls the lifecycle of the dataset being processed. This prevents dangling data references after a training session ends.

3. **Aggregation for `ModelRegistry` ↔ `MLModel`** — Models can exist in memory before being persisted, so aggregation (not composition) is appropriate. The registry is a storage layer, not an owner.

4. **`secretKey` in `AuthManager`** — Required for JWT token signing; kept private to prevent external access.

5. **`qualityScore` in `Dataset`** — Tracks data quality (missing values, outliers) as part of the ingestion pipeline; exposed via `getQualityScore()` for reporting.

6. **`shapValues` and `featureNames` in `Explainer`** — Cached after `computeSHAP()` is called to avoid recomputation when `generateReport()` and `getRecommendations()` are invoked separately.

7. **`listModels()` in `ModelRegistry`** — Supports the prediction endpoint that needs to enumerate available trained models before loading one.
